Task 1 Phase 1

A 登入 : 使用者可以以不同的身份登入，使用者身份有
	1. QA
	2. RD

B 列表 : 使用者登入後可以看到全部項目的列表，有以下欄位
	1. summary, 文字欄位 必填
	2. description, 文字欄位 必填
	3. status, enum 欄位 有 open 和 resolved 可選
	4. type enum 欄位 有 bug  可選
	

C 新增 : 依使用者身份不同，可以進行不同的新增動作
	1. QA : 可以新增 Bug
	2. 新增 Bug 時 summary 為必填欄位
	3. 新增 Bug 時 description 為必填欄位

D 變更 : 依使用者身份不同，可以進行不同的變更動作
	1. QA 可以編輯 Bug 的 summary 欄位
	2. QA 可以編輯 Bug 的 description 欄位
	3. QA 不能編輯 Bug 的status 欄位 至 Resolve 
	4. RD 可以編輯 Bug 的status 欄位 至 Resolve

E 刪除 : 依使用者身份不同，可以進行不同的刪除動作
	1: QA 可以刪除 Bug

Task 1 Phase 2

A 登入 : 增加以下的使用者身份可以登入，使用者身份有
	1. PM
	2. Administrator

B 列表 :  使用者登入後可以看到全部項目的列表，增加或修改以下欄位
	1. type enum 欄位 有 bug , feature, testcase 可選
	2. Severity, 數字欄位
	3. Priority, 數字欄位
	
C 新增 : 依使用者身份不同，可以進行不同的新增動作
	1. PM 可以新增 Feature
	1. Administrator 可以新增 Feature
	2. QA 可以新增 TestCase
	2. Administrator 可以新增 TestCase 
	2. Administrator 可以新增 Bug 

D 變更 : 依使用者身份不同，可以進行不同的變更動作
	1. QA 可以變更 TestCase 所有欄位
	2. RD 可以編輯 Feature 的status 欄位 至 Resolve
	3. PM 可以編輯 Feature 的 summary 欄位
	4. PM 可以編輯 Feature 的 description 欄位
	5. PM 可以編輯 Feature 的 Severity 欄位
	6. PM 可以編輯 Feature 的 Priority 欄位
	7. Administrator 可以變更 Bug 所有欄位
	8. Administrator 可以變更 Feature 所有欄位
	9. Administrator 可以變更 TestCase 所有欄位
	
E: 人員管理 Administrator身份使用者可以進行以下管理
	1. 新增 QA
	2. 新增 RD
	3. 新增 PM
